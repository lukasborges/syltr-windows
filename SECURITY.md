# Security policy

## Reporting a vulnerability

Please do not disclose vulnerabilities, credentials, session data or exploit
details in a public issue. Use GitHub's private vulnerability reporting flow at
<https://github.com/lukasborges/syltr-windows/security/advisories/new> when it
is available. If that option is unavailable, open a public issue containing no
sensitive details and ask the maintainer to arrange a private reporting
channel.

Include the affected version, reproduction conditions, expected impact and any
suggested mitigation. Remove tokens, cookies, account identifiers and hosted
service content from logs or screenshots before sharing them.

## Supported versions

Syltr has not published its first beta release yet. Until public releases
begin, security fixes are applied to the latest revision on the `main` branch.
The supported release range will be documented here once versioned packages
are available.
